Serverless

What is/means Serverless?

There does not seem to be a simple definition (it overlaps with BaaS)

It's not a technology, it's an approach

FaaS (Function-as-a-Service) - backend service allowing writing pieces of code that can be executed on certain events

Serverless != no server, there are still physical servers used, you just do not care about the underlying infrastructure

Typical use cases: microservices architecture (serverless itself does not mean microservices, it can be monolith as well), startups, low budget projects

  • we do not need to case about server at all, we just rent server computations
  • we pay only what we actually use
  • auto-scaling (great for inconsistent traffic)
  • saves time and increase productivity
  • quick deployment and updates
  • no infrastructure setup, configuration...
  • cold start - might be up to 10s in AWS, but there's a workaround mentioned, maybe not a big issue these days
  • testing and debugging
  • security concerns
  • not good for long running processes - execution time and memory limits
  • monitoring

Lambda Function

  • stateless function to do one thing
  • runs in its own container
  • no visibility of the underlying infrastructure
  • simple back-end code - quick development

AWS Lambda free tier: 1,000,000 free requests per month, 400,000 seconds of compute time per month

AWS Lambda-Supportive Services

  • AWS Event Bridge for communication between lambda functions
  • AWS S3 to store static assets
  • AWS DynamoDB (NoSQL) for database
  • AWS Cognito for authorization
